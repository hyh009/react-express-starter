# Installation

This repository is a pnpm workspace.

Install dependencies from the repository root:

```bash
pnpm install
```

Do not install dependencies from inside an app package:

```bash
cd apps/api
pnpm install
```

Running install inside a package can create a nested lockfile such as:

```txt
apps/api/pnpm-lock.yaml
```

Nested package lockfiles should not be used in this workspace.

## Lockfile

Commit the root workspace lockfile:

```txt
pnpm-lock.yaml
```

Do not commit package-level lockfiles:

```txt
apps/*/pnpm-lock.yaml
```

The root lockfile controls dependency versions for every workspace package, including `apps/api`.

## Adding Dependencies

Add dependencies from the repository root with a workspace filter.

For API runtime dependencies:

```bash
pnpm --filter api add <package>
```

For API development dependencies:

```bash
pnpm --filter api add -D <package>
```

For Web runtime dependencies:

```bash
pnpm --filter web add <package>
```

For Web development dependencies:

```bash
pnpm --filter web add -D <package>
```

This updates the matching package `package.json` and the root `pnpm-lock.yaml`.

## Running API Scripts

Run API scripts from the repository root:

```bash
pnpm --filter api run dev
pnpm --filter api run build
pnpm --filter api run lint
pnpm --filter api run test
```

Package-level `node_modules` directories may exist locally after installation. They are generated by pnpm and must not be committed.
